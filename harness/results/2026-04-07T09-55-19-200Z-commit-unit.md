```md
# Commit Quality Harness Prompt

## 목표
프론트엔드 작업의 커밋이 작업 단위로 충분히 세분화되어 있는지 자동 점검한다.

## 1차 평가 항목
1. 커밋 메시지 컨벤션 준수
- 형식: `fix(ui) : 한국어 상세 설명`
- 메시지는 작업 의도와 범위를 구체적으로 설명해야 한다.

2. 커밋 단위(Atomicity)
- 한 커밋은 하나의 목적만 다룬다.
- 과도하게 많은 파일/라인 변경은 분할 대상으로 본다.

3. UI 중심성
- `fix(ui)` 커밋은 UI/UX 관련 파일 중심이어야 하며,
  API/모킹/테스트 대규모 변경이 섞이면 감점한다.

## 출력 형식
각 커밋마다 아래를 출력한다.
- PASS/FAIL
- 실패 사유
- 분리 제안(어떤 기준으로 커밋을 쪼개면 좋은지)

최종 요약에는 아래를 포함한다.
- 전체 PASS 비율
- 가장 큰 문제 커밋 1~2개
- 다음 액션 3개
```
